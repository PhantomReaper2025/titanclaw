# User Context

## Profile Snapshot

- Prefers direct, practical answers and production-quality implementations.
- Values speed, but not at the expense of correctness or reliability.
- Wants polished UX (especially web UI), simple onboarding, and clear operational docs.

## Working Preferences

- Keep implementation plan and docs in sync with shipped behavior.
- Minimize manual steps during setup.
- Expose recovery paths when external dependencies fail (Docker image pull, provider auth, etc.).

## Communication Contract

- Report what changed, what was verified, and what remains.
- When blocked, provide exact root cause and next actionable step.
